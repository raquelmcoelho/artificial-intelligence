Lab 3: Adversarial Search Problem with Pac-Man

ENSICAEN - Computer Science Department

CS 2I1AE1: Artificial Intelligence

Lab 3: Adversarial Search Problem with Pac-Man

(This lab assignment is adapted from the course
”Intro to AI” by John DeNero and Dan Klein at Berkeley)

Duration: 1 session

Table of Contents

  1. Introduction
  2. General Instructions
  3. Exercise 1 (Easy): Minimax Algorithm With n Ghosts
  4. Exercise 2 (Medium): Alphabeta Algorithm
  5. Exercise 3 (Medium): Expectimax Algorithm
  6. Exercise 4 (Medium): Evaluation Function

In this project, you will design agents for the classic version of Pac-Man, including ghosts. Along the way, you will implement minimax, alphabeta, and expectimax search and design an evaluation function.

Introduction

The player controls Pac-Man through a maze, eating pac-dots. When all pac-dots are eaten, Pac-Man win. Two ghost enemies roam the maze, trying to catch Pac-Man. If a ghost touches Pac-Man, he loses. Near the corners of the maze are larger dots known as power pellets that provide Pac-Man with the temporary ability to eat the ghosts. When a ghost is eaten, it is regenerated in the center box. Ghosts flash white to signal that they are about to become dangerous again.


Figure 1: Classic Pac-Man's world.

General instructions

Download the following archive: Lab3.zip

This archive contains all the code and supporting files.

First, play a game of classic Pac-Man using the keyboard arrows if you are not familar with this game:

./pacman.py

Exercise 1: Minimax Algorithm With n Ghosts

Write the agent MinimaxAgentN considering an arbitrary number of ghosts. To help you write the code, the class MinimaxAgent1 provides an example assuming only one ghost. You can try our algorithm:

python pacman.py -p MinimaxAgent1 -l minimaxClassic -a depth=3 --number_of_ghosts=1
python pacman.py -p MinimaxAgent1 -l smallClassic -a depth=3 --number_of_ghosts=1
python pacman.py -p MinimaxAgent1 -l minimaxClassic -a depth=3 --number_of_ghosts=1 -q -n 10

The function getAction(self, gameState) you have to write should return the chosen action among the legalActions() (ie. East, West, North, South). It uses a GameState instance as argument which provides the following variables and methods:

class GameState

getLegalActions(agentIndex)

returns the list of possible directions for the agent among ('West', 'North', 'South', 'East').

generateSuccessor(agentIndex,action)

returns the successor state after the specified agent takes the specified action.

getNumberOfAgents()

returns the number of the agents.

The class MultiAgentSearchAgent (that MinimaxAgentN inherits) provides:

class MinimaxAgentNultiAgentSearchAgent

isTerminalNode(gameState, depth)

returns true if the current state is a terminal node; either Pac-Man wins or loses or depth = 0.

self.depth

contains the search depth given by the user.

self.evaluationFunction(gameState)

returns the utility value for the current game state (the current score).


Important: A single depth value is considered to be one Pac-Man move and all the ghosts' responses, so depth 2 will involve Pac-Man and each ghost moving two times. Pac-Man is always agent 0, and the agents move in order of increasing agent index.

Test your code on:

python pacman.py -p MinimaxAgentN -l minimaxClassic -a depth=3 --number_of_ghosts=3 --frameTime=0.5
python pacman.py -p MinimaxAgentN -l smallClassic -a depth=4 -k 3
python pacman.py -p MinimaxAgentN -l minimaxClassic -a depth=4 -k 2 -q -n 10

Try variation on depth value (-a depth=7).

Expected results

  • The minimax values of the initial state in the minimaxClassic layout are 9, 8, 7, -492 for depths 1, 2, 3 and 4 respectively. Note that your minimax agent will often win (6/10 games for us with depth 4).

  • Depth 4 should be pretty quick, but depth 5 or 6 will be slow. Don't worry, the next exercise will speed up the search somewhat.

  • On larger boards such as openClassic and mediumClassic (the default), you'll find Pac-Man to be good at not dying, but quite bad at winning. He'll often thrash around without making progress. He might even thrash around right next to a dot without eating it because he doesn't know where he would go after eating that dot. Don't worry if you see this behavior, exercise 3 will clean up all of these issues.

Exercise 2: Alphabeta algorithm

Make a new agent that uses alpha-beta pruning to explore more efficiently the minimax tree, in AlphaBetaAgent. You should see a speed-up (perhaps depth 3 alpha-beta will run as fast as depth 2 minimax).

python pacman.py -p AlphaBetaAgent -l smallClassic -a depth=3 -k 2
python pacman.py -p AlphaBetaAgent -l smallClassic -a depth=4 -k 3
python pacman.py -p AlphaBetaAgent -l minimaxClassic -a depth=7 -k 2
python pacman.py -p AlphaBetaAgent -l minimaxClassic -a depth=7 -k 2 -q -n 10

The AlphaBetaAgent minimax values should be identical to the MinimaxAgent minimax values, although the actions it selects can vary because of different tie-breaking behavior. So, the minimax values of the initial state in the minimaxClassic layout are 9, 8, 7 and -492 for depths 1, 2, 3 and 4 respectively and 2 ghosts.

Exercise 3: Expectimax Algorithm

Random ghosts are of course not optimal minimax agents, and so modeling them with minimax search may not be appropriate. Fill in ExpectimaxAgent, where your agent will no longer take the min over all ghost actions, but the expectation according to your agent's model of how the ghosts act.

You should now observe a more cavalier approach in close quarters with ghosts. In particular, if Pac-Man perceives that he could be trapped but might escape to grab a few more pieces of food, he'll at least try. 

python pacman.py -p ExpectimaxAgent -l smallClassic -a depth=3 -k 3
python pacman.py -p ExpectimaxAgent -l minimaxClassic -a depth=4 -k 2
python pacman.py -p ExpectimaxAgent -l minimaxClassic -a depth=4 -k 2 -q -n 10

Investigate the results of these two scenarios:

python pacman.py -p MinimaxAgentN -l trappedClassic -a depth=3 -q -n 10
python pacman.py -p ExpectimaxAgent -l trappedClassic -a depth=3 -q -n 10

Exercise 4: Evaluation Function

Write a better evaluation function for Pac-Man in the provided function MyEvaluationFunction stub. The evaluation function should evaluate states. With a depth search of 2, your evaluation function should clear the smallClassic layout with two random ghosts more than half the time and still run at a reasonable rate.

python pacman.py -p ExpectimaxAgent -l minimaxClassic -a evalFn=MyEvaluationFunction,depth=2 -k 3
python pacman.py -p ExpectimaxAgent -l minimaxClassic -a evalFn=MyEvaluationFunction,depth=2 -k 3 -q -n 100 (our score 57/100)
python pacman.py -p AlphaBetaAgent -l smallClassic -a evalFn=MyEvaluationFunction,depth=4 -k 3
python pacman.py -p ExpectimaxAgent -l minimaxClassic -a evalFn=MyEvaluationFunction,depth=4 -k 3 -q -n 10 (our score 84/100)
python pacman.py -p ExpectimaxAgent -l mediumClassic -a evalFn=MyEvaluationFunction,depth=3 -k 3 (our score 1863)

Hints: One way you might want to write your evaluation function is to use a linear combination of features.

List of useful methods and variables accessible via the class GameState:

class GameState

isWin()

returns true if Pac-Man has eaten all the dots.

isLose()

returns true if Pac-Man dies

getTotalFood()

returns the initial number of food dots.

getNumberOfFood()

returns the current number of food dots (not eaten).

getTotalGhosts()

returns the number of ghosts.

getNumberOfKilledGhosts()

returns the current number of ghosts killed by Pac-Man.

getTotalCapsules()

returns the initial number of capsules.

getNumberOfCapsules()

returns the current number of capsules.
