In this lab, you will experiment with Q-learning for reinforcement learning. You will test your agents first on Gridworld, then apply them to the task of teaching Pacman to play.


Pacman seeks reward.
Should he eat or should he run?

General Instructions

Download the archive lab4.zip in your file system.

This archive contains all the code and data for the lab. As usual, the file README.md lists all the command needed for this lab.

Exercise 1: Q-Learning Pacman

Time to use Q-learning to play some Pac-Man!

As a reminder from the lecture, the Q-learning algorithm works on the running average principle, and the update formula looks as follows if s is the current state and s' the next state using action a:

Q0(s,a) 0
Q k+1(s,a) (1-) Qk(s,a) + ( R(s) + maxa Qk(s,a) )

Fill in the code of the class QLearningAgent in the file qlearningAgents.py. You must implement the computeActionFromQValues(), computeValueFromQValues() , and update() methods:

class QLearningAgent

update(self, state, action, nextState, reward)

You should do your Q-Value update here. No return value expected.

Hint: We supply a setter and getter for accessing QValues:

self.getQValue(state, action)
self.setQValue(state, action, value)

Important: Make sure that in your method, you only access Q values by calling getQValue(). This abstraction will be useful in exercise 3.

Use the following function computeUtilityFromQValues.

The QLearningAgent class has variables that it inherits from the ReinforcementAgent class:

  • self.alpha: the learning rate [0,1].

  • self.gamma: the discount factor [0,1].

computeUtilityFromQValues(self, state)

Returns maxaction Q(state,action) where the max is over legal actions from the state. Note that if there are no legal actions, which is the case at the terminal state, you should return a value of 0.0.

Important: Make sure that in your method, you only access Q values by calling getQValue(). This abstraction will be useful for exercise 3.

A useful method is:

  • self.getLegalActions(state): returns the list of legal actions for a state.

computeActionFromQValues(self, state)

Computes the best action to take in a state among the legal ones. Note that if there are no legal actions, which is the case at the terminal state, you should return None.

Note: You should break ties randomly for better behavior. For this, the method random.choice(list) is useful, as it takes a list of elements as an argument and returns a random one with uniform probability.

Run the following command with the default values (alpha=0.2, gamma=0.8) for your tests:

python3 pacman.py -p PacmanQAgent -x 2000 -n 2010 -l smallGrid

Pacman will play games in two phases. In the first phase, training, Pacman will begin to learn about the values of positions and actions. Because it takes 2000 games to learn accurate q-values, Pacman's training games run in quiet mode by default, with no GUI display. Once Pacman's training is complete, he will enter testing mode. With the previous command, Pac-Man is tested using the last 10 games (ie, 2010 - 2000). When testing, Pacman's self.epsilon and self.alpha will be set to 0.0, effectively stopping q-learning and disabling exploration, in order to allow Pacman to exploit his learned policy.

You should observe that after training, the pacman succeeds most of the time on this maze.

If you want to experiment with learning parameters, you can use the option -a, for example -a alpha=0.3,gamma=0.7.

Exercise 2: Dilemma exploration vs exploitation

The previous agent is a greedy learner. Complete your q-learning agent by implementing -greedy action selection in getAction(), meaning it chooses random actions of the time (exploration), and follows its current best q-values otherwise (exploitation).

The QLearningAgent class has variables that it inherits from the ReinforcementAgent class:

  • self.epsilon: the exploration probability [0,1].

Hint: You can simulate a binary variable with probability p of success by using util.flipCoin(p), which returns True with probability p and False with probability 1-p.

Run your new agent (with default value of epsilon=0.05):

python3 pacman.py -p PacmanQAgent -x 2000 -n 2010 -l smallGrid

Once again, if you want to experiment with learning parameters, you can use the option -a, for example -a epsilon=0.1.

Run your new agent (with default value of epsilon=0.05):

python3 pacman.py -p PacmanQAgent -x 2000 -n 2010 -l mediumGrid

With this implementation, you will find that training the same agent on the seemingly simple mediumGrid may not work well. Pacman fails to win on larger layouts because each board configuration is a separate state with separate q-values. He has no way to generalize that running into a ghost is bad for all positions. Obviously, this approach will not scale.

Exercise 3: Approximate Q-learning and State Abstraction

To enhance your Pac-Man performance, you will implement an approximate q-learning agent that learns weights for features of states, where many states might share the same features.

In the file qlearningAgents.py, complete the implementation of the ApproximateQAgent class as follows:

  1. In getQValue(), the approximate version of the q-value takes the following form:

    Q(a, s) = i{1,n} fi(s, a) wi

    where each weight wi is associated with a particular feature fi(s,a). The dictionary self.weights is defined in the constructor as a util.Counter,  and self.featExtractor.getFeatures(state, action) is the vector of the features. Implement this as the dot product of the weights and the features.

    Code snippet that can help you, where features is a dictionary: 

    features = self.featExtractor.getFeatures(state, action)
for f in features:
   self.weights[f] = 0

  2. In update() you will change the weights similarly to how you updated q-values (cf. exercise 1):

    wi wi + [difference]fi(s,a)

    difference = (R(s,a) + maxa' Q(s',a'))-Q(s,a)

The functions that will calculate the features are already defined and implemented in the file featureExtractors.py, and the class variable self.featExtractor which is available in your ApproximatedAgent has a method getFeatures(state, action) which returns a util.Counter with feature values for a given Q-state.

Note: In order to keep weights in a reasonable range, all of these features are divided by 10.0.

Your approximate q-learning agent with the SimpleExtractor should learn to win with ease on the mediumGrid:

python3 pacman.py -p ApproximateQAgent -a extractor=SimpleExtractor -x 50 -n 60 -l mediumGrid

While with a bit longer training time, the learned Pac-Man controller should have no more problems with a larger one:

python3 pacman.py -p ApproximateQAgent -a extractor=SimpleExtractor -x 50 -n 60 -l mediumClassic

Exercise 4: Improving the feature extractor

When watching your trained approximate q-learning agent, you probably noticed that he actively avoids ghosts, even when they are safe to eat. This is because the given feature extractor defined in the class SimpleExtractor in file featureExtractors.py doesn't distinguish between safe and dangerous ghosts, so ghosts always look dangerous. If you try your approximate q-learning agent on some of the hardest layouts like contestClassic and trickyClassic, the fact that Pacman is not seeking out and using the capsules and not eating safe ghosts hurts his performance.

The given feature extractor SimpleExtractor depends on just four features:

  • "bias" always 1.0.

  • "#-of-ghosts-1-step-away" the number of ghosts (regardless of whether they are safe or dangerous) that are 1 step away from Pacman.

  • "closest-food" the distance in Pacman steps to the closest food pellet (does not take into account walls that may be in the way).

  • "eats-food" either 1 or 0 if Pacman will eat a pellet of food by taking the given action in the given state.

At the end of the file featureExtractors.py fill in the class BetterExtractor that will improve on SimpleExtractor (base your work on the class SimpleExtractor). You might want to consider incorporating:

  • The distance to the nearest capsule (use method state.getCapsules()).

  • The number of safe ghosts that are one step away. Each ghost state maintains a variable called state.getGhostState(index).scaredTimer, where index if one of the value of the list returned by state.getGhostPositions(). This timer is 0 when a ghost is dangerous to Pacman and greater than 0 when a ghost is safe for Pacman to eat.

  • The number of dangerous ghosts that are one step away.

Test your better extractor on the hardest Pacman layouts and see if it fares better than the simple one:

 python3 pacman.py -p ApproximateQAgent -a extractor=BetterExtractor -x 50 -n 60 -l contestClassic
